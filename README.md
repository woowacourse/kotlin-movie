# 🚀 1단계 - 영화 예매

## 1. 기능 요구 사항

### 1.1 상영 편성

- 하나의 영화는 상영 가능 기간 내에서 하루 여러 차례 상영될 수 있다.
- 상영 시간은 영화 상영 길이와 상영관 운영 시간을 기준으로 결정된다.
- 하나의 영화가 여러 상영관에서 동시에 상영되는 경우는 고려하지 않는다.

### 1.2 예매

- 사용자는 영화와 상영 시간을 선택한 뒤 좌석을 선택한다.
- 여러 영화를 한 번에 예매할 수 있다.
- 한 번에 예매하는 상영들 간 시간이 겹치면 함께 예매할 수 없다.

### 1.3 좌석

- 좌석 표기는 행(알파벳) + 열(숫자) 형식이다. 예: A1, C3, E4
- 이미 예약된 좌석은 다시 선택할 수 없다.
- 좌석 등급별 가격이 다르다.

### 1.4 좌석 등급 가격

| 등급 | 가격 |
  |---|---:|
| S석 | 18,000원 |
| A석 | 15,000원 |
| B석 | 12,000원 |

### 1.5 할인 및 결제

- 무비데이 할인: 매월 10일, 20일, 30일 상영 시작 건은 10% 할인
- 시간 할인: 오전 11시 이전 또는 오후 8시 이후 시작 상영은 2,000원 할인
- 포인트: 예매 금액에서 직접 차감
- 결제 수단 할인

  | 수단 | 할인율 |
  |---|---:|
| 신용카드 | 5% |
| 현금 | 2% |

### 1.6 할인 적용 순서

1. 무비데이 할인(정률)
2. 시간 할인(정액)
3. 포인트 차감
4. 결제 수단 할인(정률)

## 2. 프로그래밍 요구 사항

- 콘솔 입출력은 구현하지 않는다.
- 모든 기능은 TDD로 구현하고 단위 테스트를 작성한다.
### 객체 지향 생활 체조 원칙을 따른다.
- 한 메서드에 오직 한 단계의 들여쓰기만 사용한다.
- else 예약어를 사용하지 않는다.
- 모든 원시값과 문자열을 포장한다.
- 한 줄에 점(.)을 하나만 사용한다.
- 축약을 지양하고 의도를 드러내는 이름을 사용한다.
- 모든 엔티티를 작게 유지한다.
- 인스턴스 변수 3개 초과 클래스를 지양한다.
- 일급 컬렉션을 사용한다.
- getter/setter/properties 사용을 최소화하고 객체가 스스로 행동하게 만든
  다.

영화관(MovieTheater)

# 테스트 시나리오 (기능 요구사항 1)

### 영화/상영 생성

- Movie는 제목, 상영시간(시작, 끝)을 가지고 있다. 
- Movie는 상영가능기간 시작일/종료일 경계 날짜에서도 상영 가능하다.
- 영화관은 같은 영화를 하루에 여러 회차로 편성할 수 있다.
- 한 영화는 중복되는 시간대에 동시 상영될 수 없다.
- 상영 가능 기간 외의 날짜로 생성하면 실패한다.

### 좌석

- 좌석은 A1, C3 같은 형식으로 다중선택이 가능하다.
- 예약 가능한 좌석을 선택하면 예약 상태로 변경된다.
- 이미 예약된 좌석은 다시 예약되지 않는다.
- 여러 좌석을 한 번에 선택하면 모두 예약 가능할 때만 성공한다.
- 좌석은 행A~E 열1~12로 고정되어있다. 그 외 좌석 선택시 실패한다.
- 좌석 A,B는 B등급/ C는 A등급/ D,E S등급으로 취급한다.

### 장바구니

- 시간이 겹치지 않는 상영 영화는 함께 담을 수 있다.
- 시간이 겹치는 상영 영화를 담으려 하면 실패한다.
- 장바구니 총액은 각 상영의 좌석 금액 합으로 계산된다.

### 할인/결제

- 매월 10일/20일/30일 상영은 무비데이 10% 할인이 적용된다.
- 11:00 이전 상영은 2,000원 할인이 적용된다.
- 20:00 이후 시작 상영은 2,000원 할인이 적용된다.
- 무비데이와 시간 할인은 동시에 적용할 수 있다.
- 동시 적용 시 순서는 무비데이 후 시간할인이다.
- 포인트는 할인 적용 후 금액에서 차감된다.
- 포인트 차감 후 결제수단 할인이 적용된다.
- 포인트와 결제
- 신용카드 결제는 5% 할인이 적용된다.
- 현금 결제는 2% 할인이 적용된다.
- 최종 결제 금액은 음수가 되지 않는다.

#### 예매 확정(Reservation)

- 결제 완료된 예매는 내역에 저장된다.
- 예매 완료 후 해당 좌석은 동일 상영에서 재선택할 수 없다.
