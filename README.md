# 1단계 - 영화 예매

## 구현할 기능 목록

### 1. Money/Point 도메인
- [x] Money 객체는 금액을 갖고 있다.
- [x] Money 객체는 더하기, 빼기를 할 수 있다.
- [x] Money 객체는 비율 계산(퍼센트 할인)을 할 수 있다.
- [x] Money 객체는 금액이 0 미만이 되지 않도록 보장한다.
- [x] 포인트 객체는 잔액을 갖고 있다.
- [x] 포인트 객체는 잔액을 차감할 수 있다.
- [x] 포인트가 남은 금액보다 클 경우 금액은 0원이 된다.

### 2. 좌석 도메인
- [x] 좌석 객체는 좌석행, 좌석열, 좌석등급을 갖는 불변 객체이다.
- [x] 좌석행은 알파벳 대문자 A~E 사이여야 한다.
- [x] 좌석열은 1~4 사이어야 한다.
- [x] 좌석 등급은 S, A, B 등급을 갖고 있다.
- [x] 등급별 좌석의 가격이 다르다.(S: 18,000원, A: 15,000원, B: 12,000원)
- [x] 예약된 좌석들 객체는 예약 가능 여부를 판단한다. 
- [x] 예매 완료 시 예약된 좌석들 객체에 해당 좌석을 추가할 수 있다.
- [x] 선택 좌석들 객체는 선택한 좌석들의 총 원가를 계산할 수 있다.
- [x] 좌석 목록 객체는 상영관 전체 좌석 구성을 갖는다.
- [x] 좌석 목록 객체는 특정 좌석이 존재하는지 판단할 수 있다.

### 3. 상영 도메인
- [x] 상영관 객체는 상영관 ID와 좌석 목록을 갖고 있다.
- [x] 상영일시 객체는 날짜, 시작 시간, 끝시간을 갖고 있다.
- [x] 상영일시 객체는 무비데이인지 판단할 수 있다.
- [x] 상영일시 객체는 시간할인 대상인지 판단할 수 있다.
- [x] 상영 객체는 영화 ID, 상영관, 상영일시, 예약된 좌석들을 갖고 있다.
- [x] 상영 객체는 예약 가능 여부를 판단할 수 있다.
- [x] 상영 목록 객체는 상영 객체들을 갖는다.

## 4. 예매 도메인
- [x] 예매 객체는 상영과 선택 좌석들을 갖고 있다.
- [x] 예매 객체는 선택한 좌석들의 총 가격을 계산할 수 있다.
- [x] 예매 목록 객체는 예매 시간이 겹치는지 검증할 수 있다.
- [x] 예매 목록 객체는 예매를 목록에 추가할 수 있다.
- [x] 예매 목록 객체는 전체 예매 원가를 계산할 수 있다.

## 5. 할인 도메인
- [x] 할인 정책에는 10/20/30일에 10%를 할인해주는 무비데이 할인이 있다.
- [x] 할인 정책에는 오전 11시 전과 오후 8시 이후에 2,000원을 할인해주는 시간 할인이 있다.
- [x] 할인 정책들은 여러 할인이 적용될 경우 순서대로 할인을 적용할 수 있다.

## 6. 결제 도메인
- [x] 결제수단 객체는 5%를 할인해주는 신용카드가 있다.
- [x] 결제수단 객체는 2%를 할인해주는 현금이 있다.
- [x] 금액 계산은 전체 예매 원가 산출 -> 할인 정책 -> 포인트 차감 -> 결제 수단 할인 -> 최종 결제 금액 흐름으로 결제를 진행한다.
- [x] 결제 객체는 최종 결제 금액과 사용한 포인트 금액을 갖고 있다.

## 7. 사용자 도메인
- [x] 사용자 객체는 예매 목록과 포인트를 갖고 있다.

## 8. 영화 도메인
- [x] 영화 객체는 영화 ID와 제목, 상영 목록을 갖고 있다.
- [x] 영화 목록 객체는 제목으로 영화를 찾을 수 있다.

## 구현 전략
TDD 방식으로 구현하되 의존성이 없는 안쪽 도메인부터 바깥쪽으로 좌석 → 상영 → 영화 → 할인 → 예매 → 결제 순서로 구현한다.

# 2단계 영화 예매(콘솔)

## 구현할 기능 목록

### 1. 데이터 초기화
- [x] 영화, 상영관, 상영 정보를 초기 데이터로 구성한다.
- [x] 영화 목록: F1 더 무비, 토이 스토리, 아이언맨
- [x] 각 영화별 상영 날짜와 시간 목록을 갖는다.
- [x] 사용자의 보유 포인트를 초기화한다.

### 2. 예매 시작
- [x] 프로그램 시작 시 "새 예매를 생성하시겠습니까? (Y/N)" 메시지를 출력한다.
- [ ] Y 입력 시 예매 흐름을 시작한다.
- [ ] N 입력 시 프로그램을 종료한다.
- [ ] Y/N 외의 입력에 대해 다시 입력을 받는다.

### 3. 영화 선택
- [x] "예매할 영화 제목을 입력하세요" 메시지를 출력한다.
- [ ] 사용자가 입력한 영화 제목으로 영화를 조회한다.
- [ ] 존재하지 않는 영화를 입력하면 다시 입력을 받는다.

### 4. 날짜 선택
- [x] "날짜를 입력하세요 (YYYY-MM-DD)" 메시지를 출력한다.
- [ ] 해당 날짜에 상영이 없으면 다시 입력을 받는다.

### 5. 상영 시간 선택
- [ ] 해당 날짜의 상영 목록을 번호와 함께 출력한다.
- [ ] 사용자가 상영 번호를 선택한다.
- [ ] 기존 장바구니의 예매와 시간이 겹치면 "선택하신 상영 시간이 겹칩니다" 메시지를 출력하고 다시 선택을 받는다.
- [ ] 유효하지 않은 번호를 입력하면 다시 입력을 받는다.

### 6. 좌석 선택
- [ ] 좌석 배치도를 출력한다. (행, 열, 등급 표시)
- [ ] 이미 예약된 좌석은 [XX]로 표시한다.
- [x] "예약할 좌석을 입력하세요 (A1, B2)" 메시지를 출력한다.
- [ ] 쉼표로 구분된 좌석 입력을 파싱한다.
- [ ] 이미 예약된 좌석을 선택하면 다시 입력을 받는다.
- [ ] 존재하지 않는 좌석을 입력하면 다시 입력을 받는다.

### 7. 장바구니 추가
- [x] 예매 정보를 장바구니에 추가하고 확인 메시지를 출력한다.
- [x] "다른 영화를 추가하시겠습니까? (Y/N)" 메시지를 출력한다.
- [ ] Y 입력 시 2번(영화 선택)부터 반복한다.
- [ ] N 입력 시 장바구니 전체 내역을 출력한다.

### 8. 포인트 입력
- [x] "사용할 포인트를 입력하세요 (없으면 0)" 메시지를 출력한다.
- [ ] 보유 포인트보다 큰 값을 입력하면 다시 입력을 받는다.
- [ ] 숫자가 아닌 값을 입력하면 다시 입력을 받는다.

### 9. 결제 수단 선택
- [x] 결제 수단 목록을 출력한다. (1: 신용카드 5% 할인, 2: 현금 2% 할인)
- [ ] 사용자가 결제 수단을 선택한다.
- [ ] 유효하지 않은 번호를 입력하면 다시 입력을 받는다.

### 10. 결제
- [ ] 최종 결제 금액을 계산하여 출력한다.
- [ ] 원가 → 할인정책(무비데이, 시간) → 포인트 차감 → 결제수단 할인
- [ ] "위 금액으로 결제하시겠습니까? (Y/N)" 메시지를 출력한다.
- [ ] Y 입력 시 예매 완료 내역을 출력하고 프로그램을 종료한다.
- [ ] N 입력 시 프로그램을 종료한다.

### 11. 출력 형식
- [ ] 상영 목록은 "[번호] 시작시간" 형식으로 출력한다.
- [ ] 좌석 배치도는 행(A~E), 열(1~4), 등급(S/A/B)을 표시한다.
- [ ] 장바구니 항목은 "[영화제목] 날짜 시간 좌석: 좌석목록" 형식으로 출력한다.
- [ ] 최종 금액은 천 단위 콤마를 포함하여 출력한다.
