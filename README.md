# kotlin-movie

# 🚀 1단계 - 영화 예매

## 1. 기능 요구 사항

### 1.1 상영 편성

- 하나의 영화는 상영 가능 기간 내에서 하루 여러 차례 상영될 수 있다.
- 상영 시간은 영화 상영 길이와 상영관 운영 시간을 기준으로 결정된다.
- 하나의 영화가 여러 상영관에서 동시에 상영되는 경우는 고려하지 않는다.

### 1.2 예매

- 사용자는 영화와 상영 시간을 선택한 뒤 좌석을 선택한다.
- 여러 영화를 한 번에 예매할 수 있다.
- 한 번에 예매하는 상영들 간 시간이 겹치면 함께 예매할 수 없다.

### 1.3 좌석

- 좌석 표기는 행(알파벳) + 열(숫자) 형식이다. 예: A1, C3, E4
- 이미 예약된 좌석은 다시 선택할 수 없다.
- 좌석 등급별 가격이 다르다.

### 1.4 좌석 등급 가격

| 등급 | 가격 |
  |---|---:|
| S석 | 18,000원 |
| A석 | 15,000원 |
| B석 | 12,000원 |

### 1.5 할인 및 결제

- 무비데이 할인: 매월 10일, 20일, 30일 상영 시작 건은 10% 할인
- 시간 할인: 오전 11시 이전 또는 오후 8시 이후 시작 상영은 2,000원 할인
- 포인트: 예매 금액에서 직접 차감
- 결제 수단 할인

  | 수단 | 할인율 |
      |---|---:|
  | 신용카드 | 5% |
  | 현금 | 2% |

### 1.6 할인 적용 순서

1. 무비데이 할인(정률)
2. 시간 할인(정액)
3. 포인트 차감
4. 결제 수단 할인(정률)

## 2. 프로그래밍 요구 사항

- 콘솔 입출력은 구현하지 않는다.
- 모든 기능은 TDD로 구현하고 단위 테스트를 작성한다.

### 객체 지향 생활 체조 원칙을 따른다.
- 한 메서드에 오직 한 단계의 들여쓰기만 사용한다.
- else 예약어를 사용하지 않는다.
- 모든 원시값과 문자열을 포장한다.
- 한 줄에 점(.)을 하나만 사용한다.
- 축약을 지양하고 의도를 드러내는 이름을 사용한다.
- 모든 엔티티를 작게 유지한다.
- 인스턴스 변수 3개 초과 클래스를 지양한다.
- 일급 컬렉션을 사용한다.
- getter/setter/properties 사용을 최소화하고 객체가 스스로 행동하게 만든
  다.

## 테스트 시나리오

### 영화

- [] 영화는 제목과 상영시간을 가지고 있다
- [] 영화의 상영시간은 0보다 커야 한다
- [] 영화의 시작 상영 가능일은 종료 상영 가능일보다 클 수 없다

### 일정

- [] 일정은 시작 시각과 종료 시각을 가지고 있다
- [] 일정의 총 시각(시작 시각부터 종료 시각까지)은 영화의 상영 시간보다 크거나
  같아야 한다
- [] 일정은 좌석들을 갖고 있다
- [] 일정은 선택한 좌석들을 예약할 수 있다
- [] 두 일정의 시간이 겹치는지 알 수 있다
- [] 두 일정의 시간이 겹치지 않으면 겹치지 않음을 알 수 있다

### 좌석

- [] 좌석은 행(알파벳)과 열(숫자)로 구성된 이름을 갖는다
- [] 좌석은 예약여부 정보를 갖고 있다
- [] 좌석은 등급을 가지고 있다
- [] 이미 예약된 좌석을 선택한다면 에러가 발생한다
- [] 예약되지 않은 좌석을 예약한다면 예약이 완료된 좌석이 반환된다

### 좌석들

- [] 여러 좌석의 묶음을 가지고 있다
- [] 여러 좌석을 선택하여 예매할 수 있다
- [] 존재하지 않는 좌석을 가져올 수 없다
- [] 여러 좌석의 총 금액을 계산할 수 있다

### 등급

- [] 등급은 B, A, S로 이루어져 있다
- [] 각 등급은 12,000, 15,000, 18,000원의 가격을 갖는다

### 할인혜택

- [] 매월 10, 20, 30일에 상영되는 영화는 10% 할인된다
- [] 오전 11시 이전 또는 오후 8시 이후에 시작하는 영화는 2,000원 할인된다
- [] 두 가지의 할인혜택은 동시에 적용될 수 있다
- [] 두 가지의 할인혜택이 적용될 때에는 퍼센트 할인이 먼저 적용된다
- [] 포인트 할인 시 예매 금액에서 포인트만큼의 금액이 차감된다
- [] 포인트는 예매 금액보다 크게 사용할 수 없다
- [] 신용카드로 결제 시 5% 할인된다
- [] 현금으로 결제 시 2% 할인된다
- [] 결제방식 할인은 포인트 할인이 적용된 후에 적용된다

### 장바구니

- [] 장바구니에 예매 항목을 추가할 수 있다
- [] 여러 영화를 장바구니에 담을 수 있다
- [] 장바구니에 담긴 모든 항목의 총 금액을 계산할 수 있다
- [] 총 금액 계산 시 항목별로 무비데이·시간 할인이 각각 적용된다
- [] 총 금액 계산 시 포인트 할인은 합산 금액에 한 번만 적용된다
- [] 총 금액 계산 시 결제 수단 할인은 포인트 할인 후 금액에 적용된다
