## 기능 목록

### 좌석
- [ ] 열이 한 글자 대문자 알파벳인지 확인한다.
  - [ ] 해당 형식이 아닐 경우, "열은 한 글자 대문자 알파벳이여야 합니다."라는 에러 문구를 출력한다.
- [ ] 행이 양수인지 확인한다.
  - [ ] 해당 형식이 아닐 경우, "행은 양수이여야 합니다."라는 에러 문구를 출력한다.

### 사용자
- [ ] 사용 여부에 따라 포인트 차감한다.
- [ ] 입력된 ID는 양수인지 확인한다.
  - [ ] 해당 형식이 아닐 경우, "ID는 양수이어야 합니다."라는 에러 문구를 출력한다.

### 금액
- [ ] 조건에 따른 할인(할인정책에서의 무비데이, 시간 조건 순)을 연산한다.
- [ ] 포인트는 예매 금액에서 직접 차감된다.
- [ ] 결제 수단에 따라 할인 정책을 연산한다.

### 할인정책
- [ ] 무비데이 조건: 매월 10일, 20일, 30일에 상영되는 영화는 10% 할인된다.
- [ ] 시간 조건: 오전 11시 이전 또는 오후 8시 이후에 시작하는 상영은 2,000원이 할인된다.
- [ ] 신용카드 결제는 5% 할인
- [ ] 현금 결제는 2% 할인

### 예매
- [ ] 예매 영화, 예매 시간, 사용자 정보를 입력받는다.
- [ ] 예매 영화, 예매 시간, 사용자 정보 입력 무결성 확인
  - [ ] 예매 영화가 INT값이 맞는지 확인한다.
    - [ ] INT 값이 아닐 경우, "입력된 값이 유효하지 않습니다."라는 에러 문구를 반환한다.
  - [ ] 전체 영화 목록에서 해당 영화가 존재하는지 확인한다.
    - [ ] 영화가 존재하지 않을 경우, "해당 영화는 존재하지 않습니다."라는 에러 문구를 반환한다.
  - [ ] 예매 시간(영화 시작)에 대해 "0000-00-00 00:00" 형식을 확인한다.
    - [ ] 해당 형식이 아닐 경우, "입력된 값이 유효하지 않습니다."라는 에러 문구를 반환한다.
  - [ ] 입력한 예매 영화에 대해서 예매 시간에 해당하는 상영 시간이 존재하는지 확인한다.
    - [ ] 입력된 예매 시간이 잘못되었을 경우 "해당 영화는 해당 시간에 상영되지 않습니다."라는 에러 문구를 출력한다.
  - [ ] 입력한 사용자 정보가 전체 사용자 정보에 존재하는지 확인한다.
    - [ ] 사용자 정보가 존재하지 않을 경우, 전체 사용자 정보에 해당 사용자 정보를 추가한다.
  - [ ] 예매한 시간에 대해서 해당 사용자가 예매한 내역 중에 시간이 겹치는 정보가 존재하는지 확인한다.
    - [ ] 존재할 경우에는 "동일한 시간대에 예매한 내역이 존재합니다."라는 에러 문구를 출력한다.
- [ ] 좌석 정보를 입력받는다.
- [ ] 좌석 입력 무결성 확인
  - [ ] 좌석 정보가 "(알파벳)(숫자)" 형식인지 확인한다.
    - [ ] 해당 형식이 아닐 경우, "입력된 값이 유효하지 않습니다."라는 에러 문구를 반환한다.
  - [ ] 해당 좌석 정보가 해당 상영관에 존재하는지 확인한다.
    - [ ] 해당 상영관에 존재하지 않을 경우, "해당 상영관에는 해당 좌석이 존재하지 않습니다."라는 에러 문구를 반환한다.
  - [ ] 해당 좌석이 예약 가능한지 확인한다.
    - [ ] 이미 예약된 좌석일 경우, "해당 좌석은 이미 예약되었습니다."라는 에러 문구를 반환한다.
- [ ] 입력한 좌석의 금액을 가져온다.
- [ ] 입력한 좌석의 금액에 대해서 특정 조건 할인을 적용한다. (할인 적용 기능은 금액에서 진행한다.)
- [ ] 사용자에게 포인트 사용 여부를 입력받는다.
  - [ ] 입력된 사용 여부의 형식은 y 혹은 N 인지 확인한다.
  - [ ] 해당 형식이 아닐 경우, "사용 여부는 y 혹은 N으로 입력해야 합니다."라는 에러 문구를 출력한다.
- [ ] 포인트 사용 여부가 y일 경우, 금액에서 차감하고 포인트도 차감한다.
- [ ] 사용자에게 결제 수단을 입력받는다.
  - [ ] 입력된 결제 수단은 card 혹은 cash 인지 확인한다.
  - [ ] 해당 형식이 아닐 경우, "결제 수단은 card 혹은 cash로 입력해야 합니다."라는 에러 문구를 출력한다.
- [ ] 입력된 결제 수단을 금액에 전달하여 할인을 적용한다.

## 단위 테스트 목록

### 좌석(Seat)
- [x] 열이 한 글자 대문자 알파벳이면 좌석이 정상 생성된다.
- [x] 열이 한 글자 대문자 알파벳이 아니면 예외가 발생한다.
- [x] 행이 양수이면 좌석이 정상 생성된다.
- [x] 행이 0이면 예외가 발생한다.
- [x] 행이 음수이면 예외가 발생한다.

### 좌석등급(SeatGrade)
- [x] S석의 가격은 18,000원이다.
- [x] A석의 가격은 15,000원이다.
- [x] B석의 가격은 12,000원이다.

### 사용자(User)
- [x] ID는 양수일 때 정상 생성 되어야 한다.
- [x] ID가 음수일 때 예외가 발생한다.
- [x] 포인트 사용 시 포인트가 차감된다.
- [x] 보유 포인트보다 많은 금액을 차감하면 예외가 발생한다.

### 할인정책(DiscountPolicy)
- [ ] 매월 10일에 상영되는 영화는 10% 할인된다.
- [ ] 매월 20일에 상영되는 영화는 10% 할인된다.
- [ ] 매월 30일에 상영되는 영화는 10% 할인된다.
- [ ] 무비데이가 아닌 날에는 무비데이 할인이 적용되지 않는다.
- [ ] 오전 11시 이전에 시작하는 상영은 2,000원이 할인된다.
- [ ] 오후 8시 이후에 시작하는 상영은 2,000원이 할인된다.
- [ ] 오전 11시 ~ 오후 8시 사이에 시작하는 상영은 시간 할인이 적용되지 않는다.
- [ ] 신용카드 결제 시 5% 할인된다.
- [ ] 현금 결제 시 2% 할인된다.

### 금액(Price)
- [ ] 무비데이 할인과 시간 할인이 순서대로 적용된다.
- [ ] 포인트가 예매 금액에서 직접 차감된다.
- [ ] 결제 수단에 따라 할인이 적용된다.
- [ ] 할인 전 금액, 총 할인 금액, 할인 된 가격이 올바르게 계산된다.

### 예매(Reservation)
- [ ] 예매 영화 입력이 INT 값이 아니면 예외가 발생한다.
- [ ] 존재하지 않는 영화를 예매하면 예외가 발생한다.
- [ ] 예매 시간이 "0000-00-00 00:00" 형식이 아니면 예외가 발생한다.
- [ ] 해당 영화가 해당 시간에 상영되지 않으면 예외가 발생한다.
- [ ] 존재하지 않는 사용자는 전체 사용자 정보에 추가된다.
- [ ] 동일한 시간대에 이미 예매한 내역이 있으면 예외가 발생한다.
- [ ] 좌석 정보가 "(알파벳)(숫자)" 형식이 아니면 예외가 발생한다.
- [ ] 해당 상영관에 존재하지 않는 좌석이면 예외가 발생한다.
- [ ] 이미 예약된 좌석이면 예외가 발생한다.
- [ ] 포인트 사용 여부가 y 또는 N이 아니면 예외가 발생한다.
- [ ] 결제 수단이 card 또는 cash가 아니면 예외가 발생한다.
