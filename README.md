# 1단계 - 영화 예매

## 구현할 기능 목록

### 1. Money/Point 도메인
- [x] Money 객체는 금액을 갖고 있다.
- [x] Money 객체는 더하기, 빼기를 할 수 있다.
- [x] Money 객체는 비율 계산(퍼센트 할인)을 할 수 있다.
- [x] Money 객체는 금액이 0 미만이 되지 않도록 보장한다.
- [x] 포인트 객체는 잔액을 갖고 있다.
- [x] 포인트 객체는 잔액을 차감할 수 있다.
- [x] 포인트가 남은 금액보다 클 경우 금액은 0원이 된다.

### 2. 좌석 도메인
- [x] 좌석 객체는 좌석행, 좌석열, 좌석등급을 갖는 불변 객체이다.
- [x] 좌석행은 알파벳 대문자 A~E 사이여야 한다.
- [x] 좌석열은 1~4 사이어야 한다.
- [x] 좌석 등급은 S, A, B 등급을 갖고 있다.
- [x] 등급별 좌석의 가격이 다르다.(S: 18,000원, A: 15,000원, B: 12,000원)
- [ ] 예약된 좌석들 객체는 예약 가능 여부를 판단한다. 
- [ ] 예매 완료 시 예약된 좌석들 객체에 해당 좌석을 추가할 수 있다.
- [x] 선택 좌석들 객체는 선택한 좌석들의 총 원가를 계산할 수 있다.
- 
### 3. 상영 도메인
- [ ] 상영관 객체는 상영관 ID와 좌석 목록을 갖고 있다.
- [ ] 상영일시 객체는 날짜, 시작 시간, 끝시간을 갖고 있다.
- [ ] 상영일시 객체는 영화관 운영 시간과 겹치는지 판단할 수 있다. 
- [ ] 상영일시 객체는 무비데이인지 판단할 수 있다.
- [ ] 상영일시 객체는 시간할인 대상인지 판단할 수 있다.
- [ ] 상영 객체는 영화 ID, 상영관, 상영일시, 예약된 좌석들을 갖고 있다.
- [ ] 상영 객체는 좌석을 예약할 수 있다.
- [ ] 상영 객체는 다른 상영과 시간이 겹치는지 판단할 수 있다.

## 4. 예매 도메인
- [ ] 예매 객체는 상영과 선택 좌석들을 갖고 있다.
- [ ] 예매 객체는 선택한 좌석들의 총 가격을 계산할 수 있다.
- [ ] 예매 목록 객체는 예매 시간이 겹치는지 검증할 수 있다.
- [ ] 예매 목록 객체는 예매를 목록에 추가할 수 있다.
- [ ] 예매 목록 객체는 전체 예매 원가를 계산할 수 있다.

## 5. 할인 도메인
- [ ] 할인 정책에는 10/20/30일에 10%를 할인해주는 무비데이 할인이 있다.
- [ ] 할인 정책에는 오전 11시 전과 오후 8시 이후에 2,000원을 할인해주는 시간 할인이 있다.
- [ ] 할인 정책들은 여러 할인이 적용될 경우 순서대로 할인을 적용할 수 있다.

## 6. 결제 도메인
- [ ] 결제수단 객체는 5%를 할인해주는 신용카드가 있다.
- [ ] 결제수단 객체는 2%를 할인해주는 현금이 있다.
- [ ] 금액 계산은 전체 예매 원가 산출 -> 할인 정책 -> 포인트 차감 -> 결제 수단 할인 -> 최종 결제 금액 흐름으로 결제를 진행한다.
- [ ] 결제 객체는 최종 결제 금액과 결제수단, 결제 완료 여부를 갖고 있다.

## 7. 사용자 도메인
- [ ] 사용자 객체는 예매 목록과 포인트를 갖고 있다.

## 8. 영화 도메인
- [ ] 영화 객체는 영화 ID와 상영 목록을 갖고 있다.

## 구현 전략
TDD 방식으로 구현하되 의존성이 없는 안쪽 도메인부터 바깥쪽으로 좌석 → 상영 → 영화 → 할인 → 예매 → 결제 순서로 구현한다.
