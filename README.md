# kotlin-movie

## 기능 요구 사항

한 영화관에서 쉽고 빠르게 영화를 예매할 수 있는 프로그램을 구현한다.

- 하나의 영화는 주어진 상영 가능 기간에 하루에도 여러 차례 상영될 수 있다.
- 상영 시간은 영화의 상영 길이와 상영관 운영 시간에 따라 정해진다.
- 하나의 영화가 여러 상영관에서 동시에 상영되는 상황은 고려하지 않는다.
- 여러 영화를 한 번에 예매할 수도 있다. 이때 서로 시간이 겹치는 상영은 함께 예매할 수 없다.
- 사용자는 영화와 상영 시간을 고른 뒤 원하는 좌석을 선택할 수 있다.
- 각 좌석은 행(알파벳)과 열(숫자)로 표시된다. 예를 들어 A1, C3, E4와 같이 나타낸다.
- 좌석은 등급에 따라 가격이 다르다.
  - S석은 18,000원
  - A석은 15,000원
  - B석은 12,000원
- 이미 예약된 좌석은 다시 선택할 수 없다.
- 특정한 조건을 만족하는 예매자는 추가 혜택을 받을 수 있다.
  - 무비데이 조건: 매월 10일, 20일, 30일에 상영되는 영화는 10% 할인된다.
  - 시간 조건: 오전 11시 이전 또는 오후 8시 이후에 시작하는 상영은 2,000원이 할인된다.
  - 두 조건이 동시에 적용될 수 있으며, 무비데이 할인(비율)이 먼저 적용되고, 이어서 시간 조건 할인(정액)이 적용된다.
- 사용자는 보유한 포인트를 사용할 수 있다. 포인트는 예매 금액에서 직접 차감된다.
- 포인트를 적용한 뒤 결제 수단에 따른 추가 할인이 적용된다.
  - 신용카드 결제는 5% 할인
  - 현금 결제는 2% 할인

## 구현할 기능 목록

### 1. movie

- [ ] 영화의 이름을 알 수 있다.
- [ ] 영화의 러닝 타임을 알 수 있다.
- [ ] 영화의 상영 가능 기간을 알 수 있다.

### 2. screening schedule

- [ ] 입력 받은 영화 이름으로 해당 영화의 상영 일정들을 확인할 수 있다.
- [ ] 입력 받은 영화 이름이 존재하지 않는다면 exception을 발생시킨다.
- [ ] 입력 받은 영화 이름과 상영 일정으로 상영관 정보를 확인할 수 있다.
- [ ] 입력 받은 영화 이름, 상영 일정과 일치하는 상영관 정보가 존재하지 않는다면 exception을 발생시킨다.

### 3. screen

- [ ] 상영 영화 정보를 확인할 수 있다.
- [ ] 영화의 시작 시간을 알 수 있다.
- [ ] 전체 좌석 목록을 확인할 수 있다.

### 4. seat

- [ ] 좌석의 위치를 알 수 있다.
- [ ] 좌석의 등급을 알 수 있다.
- [ ] 좌석의 예약 여부를 알 수 있다.

### 5. seat number

- [ ] seat의 행 번호를 알 수 있다.
- [ ] seat의 열 번호를 알 수 있다.

### 6. seat grade

- [ ] seat의 가격을 알 수 있다.

### 7. reservation

- [ ] 영화 이름을 알 수 있다.
- [ ] 영화 시작 시간을 알 수 있다.
- [ ] 영화 상영 일자를 알 수 있다.
- [ ] 영화 좌석 정보를 알 수 있다.
- [ ] 영화 상영관을 알 수 있다.
- [ ] 상영 시간이 서로 겹치는 영화들을 함께 예매하려고 하면 예외를 발생시킨다.

### 8. pay process

- [ ] 입력 받은 포인트 만큼 티켓 각격이 차감된다.
- [ ] reservation의 좌석 정보를 통해 할인 정책이 적용된 티켓 가격을 계산한다.

### 9. discount policy

- [ ] 예약 시간이 오전 11이전 또는 오후 8시 이후라면 2,000 할인 된다.
- [ ] 예약 날짜가 10, 20, 30일 중 하나라면 10% 할인 된다.
- [ ] 예약 시간과 날짜가 할인 정책에 해당한다면 날짜 조건이 먼저 적용되고 시간 조건이 적용된다.
- [ ] 현금 결제시 2% 할인된다.
- [ ] 카드 결제시 5% 할인된다.
- [ ] 할인 정책에 해당하지 않는다면 할인이 적용되지 않는다.

### 10. payment

### 11. point

- [ ] 포인트 잔액을 확인할 수 있다.
- [ ] 사용한 포인트만큼 보유 포인트를 차감한다.
- [ ] 보유한 포인트보다 많은 포인트를 사용하려고 하면 예외를 발생시킨다.
